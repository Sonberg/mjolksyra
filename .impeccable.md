# Mjolksyra — Design Context

## Design Context

### Users
Hobby and professional fitness coaches managing a roster of athletes. They use Mjolksyra at a desk — planning workout blocks, reviewing athlete progress, and delivering feedback. The context is focused work: building programs, reading data, making adjustments. Athletes are secondary users who view and track assigned sessions. Both audiences are goal-oriented; they come to get something done, not to browse.

### Brand Personality
**Precise. Driven. Efficient.**

Mjolksyra (lactic acid) is about the point where effort is real — not polished, not decorative. The brand should feel like a tool built by someone who takes training seriously, not a consumer wellness app. Content and data are the product. The interface should disappear and let the work surface.

### Aesthetic Direction
- **Primary theme**: Dark. Coaches work evenings and in gym environments. Dark mode is the native state.
- **Sharp corners**: No rounding. Zero or near-zero border-radius everywhere. Precision over softness.
- **Reference**: shadcn component quality and density — content-first, neutral, unsentimental — but with sharper geometry. The components should feel like precision instruments, not friendly forms.
- **Color**: Near-monochromatic dark palette. The existing zinc-based dark tokens are correct. No bright accents. If an accent is needed, it should be barely-there — a slightly warm off-white or a restrained, single hue used sparingly.
- **AI integration**: Automation features should feel native and woven in, not bolted on. No "✨ AI" badges or sparkle icons. AI surfaces data and suggestions within the flow, like a clipboard, not a chatbot.
- **Anti-references**: Consumer fitness apps (Strava, MyFitnessPal aesthetic), rounded card grids, gradient CTAs, friendly onboarding illustrations, neon accents on dark.

### Design Principles
1. **Content is the interface** — every pixel that isn't data or a control is a distraction. Remove it.
2. **Sharp geometry, no softening** — border-radius at 0 or 2px maximum. Precision signals discipline.
3. **Dark as default, not as style** — the dark palette is chosen for context, not aesthetics. It should be readable and long-session comfortable, not dramatic.
4. **AI is invisible infrastructure** — automation features don't announce themselves; they accelerate the existing workflow silently.
5. **Density over spaciousness** — coaches want to see more data, not more whitespace. Tight spacing, clear hierarchy, no padding inflation.

### Typography Notes
- Geist is appropriate and should be retained — it's clean, neutral, and reads well at small sizes in dense UIs.
- Heading weight should be semibold (600), body at 400, labels at 500 — no decorative display fonts needed for app UI.
- Numeric data (reps, sets, weights, dates) should use tabular-nums (`font-data` utility class already exists).
- All-caps labels with `tracking-[0.15em]` at `text-[10px]` are established and correct for section eyebrows.

### Spacing & Tokens
- Tight default: `p-2` to `p-3` for component interiors, `gap-1.5` to `gap-2` between dense elements.
- Use existing `--shell-*` token system; don't introduce new hard-coded values.
- Shell design system radius tokens: `--mjolksyra-radius-sm: 0.125rem` is the maximum to use. Default to 0 where possible.
